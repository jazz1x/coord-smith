### 2.3 FINAL_STOP Root Cause Analysis (New)

**관측 사실**:
- `docs/product/rag-archive.json:570` durable lesson: "avoid scope-widening-by-attrition via endless micro-tests" — 과거에 동일 증상을 기록.
- `docs/llm/low-attention-coverage-ledger.json` 에 `FINAL_STOP — Phase R32 complete` / R33 / R34 / … / R42+ 가 순차 기록 — 각 FINAL_STOP 선언 **직후 다음 Phase 가 시작됨** = 정지 규약 무시.
- 현 `work-rag.json.current` 에 `next_action: "Phase R91 heuristic gap scan"` 과 `goal: "... FINAL_STOP reached"` 가 **동시에 존재** — 두 필드 간 모순.

**근본 원인 (3층)**:

1. **데이터 층**: `work-rag.json` 스키마가 `next_action` 과 FINAL_STOP 상태의 **상호 배타성을 제약으로 걸고 있지 않음**. 두 값이 병존 가능한 스키마 자체가 모순의 원천.
2. **코드 층**: `autoloop_runner` 가 `next_action` 을 읽어 LLM 에 전달할 때 `FINAL_STOP — *` 패턴을 **hard halt 로 해석하지 않음** (가드 부재 가능성). LLM 은 `next_action` 의 phase-이름 문자열을 action-able 로 받아들임.
3. **프롬프트 층**: 자율-루프 프롬프트가 "find next uncovered clause / extend coverage" 편향을 가져, FINAL_STOP 조건에서도 LLM 이 새 micro-phase 를 생성.

**결과**: 본 PRD 가 `work-rag.json` 문자열만 교체해도 **2, 3 층을 고치지 않으면 R92 에서 같은 패턴 반복**.

**이 PRD 가 취할 조치**:
- (a) 데이터: `next_action` 단일 패턴 규약 (§1.2) + schema-level 주석 추가 (`docs/product/work-rag.json` 상단).
- (b) 코드: `autoloop_runner.py` 에 **`^FINAL_STOP — ` 정규식 매치 시 LLM 호출 전 조기 반환** 가드 (§3.1 #19, 없을 때만 추가).
- (c) 프롬프트: `docs/llm/repo-autonomous-loop-adapter.yaml` 및 `docs/llm/agents/contract-scope-guardian.md` 의 프롬프트/규약에 **"FINAL_STOP 은 hard halt, 새 phase 생성 금지"** 문구 추가.

### 2.4 Sequencing Strategy (New)

17개 파일 변경을 **3 tier** 로 분류. 각 tier 는 **이전 tier 완료 후 착수**. tier 내부는 병렬 가능.

| Tier | 파일 수 | 특성 | 우선순위 사유 |
|------|--------|------|---------------|
| **P0 (Blocking fix)** | 4 | 자율 에이전트가 직접 소비 | FINAL_STOP 재발 방지의 선행 조건 |
| **P1 (Canon truth)** | 7 | PRD/메모리의 ceiling 표기 | SOT 캐논 정렬 |
| **P2 (Ambient polish)** | 6 | 헤더/전환 스냅샷/온보딩 | 캐논이 맞은 뒤의 정합 마감 |

P0 4개를 먼저 끝내지 않으면 P1/P2 작업 중 autoloop 가 다시 FINAL_STOP 을 덮어쓸 위험.

### 3.3 `rag.json` Revision Methodology (New)

`rag.json` 은 durable memory 이므로 단순 덮어쓰기 금지. 규약:

1. **In-place edit**: 대상 `lesson_entry` 의 `lesson` 텍스트를 최신 캐논으로 교체.
2. **Metadata bump**: 동일 entry 의 `recorded_at` 를 개정 시점으로 갱신. `tags` 에 `"ceiling-expansion-runCompletion"` 추가.
3. **Archive append**: 개정 **직전 원본 lesson_entry** 를 `docs/product/rag-archive.json` 배열 말미에 append. 새 필드 `"archived_at": "2026-04-23T00:00:00Z"`, `"archive_reason": "ceiling expanded pageReadyObserved→runCompletion 2026-03-26"` 추가.
4. **Canonical layer rule**: 이 과정에서 `rag-archive.json` 은 캐논 메모리 계층으로 승격되지 않음 — 압축 보관 아티팩트 지위 유지 (Hard prohibition §6).
5. **Atomic**: in-place edit 과 archive append 를 **동일 커밋**에 묶어 history 무결성 유지.
