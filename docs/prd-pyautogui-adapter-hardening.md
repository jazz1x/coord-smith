## 1. Planning Summary

### 1.1 Core Requirements

- `src/ez_ax/adapters/pyautogui_adapter.py` 의 silent-failure 제거로 **evidence
  무결성** 보장.
- 권한/환경 실패를 typed exception 으로 전환 — 현재는 blanket `except
  Exception: return` 으로 은폐되어 `evidence_refs` 가 거짓 참조를 돌려줌.
- `click(x, y)` 후 cursor-position 라운드트립으로 실제 실행 검증.
- 런타임 시작 시 **1회 preflight**: `moveTo(현재)` 라운드트립 + `screenshot()` 1회
  시도. 실패 시 즉시 중단.
- 기존 746 passed 무수정 유지. 권한 의존 테스트는 `pytest -m real` 마커로 분리.

### 1.2 Success Metrics

- `uv run pytest -q` ⇒ **748 passed, 1 skipped** (기존 746 + 신규 mocked 2).
- `uv run pytest -m real -q` 는 권한 있는 로컬에서 **그린**, CI 에서는 `-m "not real"` 로 자동 skip.
- `grep -n "except Exception" src/ez_ax/adapters/pyautogui_adapter.py` = **0**.
- 권한 없는 환경에서 `pyautogui_cli_entrypoint.main()` 호출 시 exit code ≠ 0, stderr 에 명확한 preflight 실패 메시지.
- evidence_refs 에 `evidence://screenshot/<key>` 가 포함되면 해당 파일이 **실재**해야 함 (적분 테스트로 검증).
- `README.md` / `AGENTS.md` 에 macOS Accessibility + Screen Recording 권한 요구 명시.

### 1.3 Key User Flows

1. **권한 있는 환경**: preflight 통과 → 12 미션 정상 실행 → evidence 파일 실재 → 정상 종료.
2. **Accessibility 권한 없음**: preflight 의 `moveTo` 라운드트립 실패 → `AccessibilityPermissionDenied` 에러 → exit 시작 단계에서 중단.
3. **Screen Recording 권한 없음**: preflight 의 `screenshot()` 실패 → `ScreenCapturePermissionDenied` → 중단.
4. **실행 중간 screenshot 실패**: 해당 미션에서 `ScreenCaptureUnavailable` 발생 → evidence_ref 거짓 반환 대신 FlowError 로 승격.
5. **좌표가 screen bounds 밖**: `ClickCoordinatesOutOfBounds` 에러 → 해당 미션 실패.

### 1.4 Non-Functional Requirements

- **결정성 유지**: 런타임 그래프 LLM 호출 금지 (기존 불변식).
- **권한 경계 유지**: OpenClaw = browser-facing, ez-ax = orchestration. adapter 는 OS 레벨만 만짐.
- **Released ceiling**: `runCompletion` 유지. 본 PRD 는 범위 변경 없음.
- **Two-layer memory**: 변동 없음.
- **테스트 성능**: preflight 는 1회성, 실행시간 영향 미미 (< 500ms on macOS).

### 3.2 Component/Function Changes

**`src/ez_ax/models/errors.py`** — 신규 5 예외. `ExecutionTransportError` 상속. 각 1줄 docstring.

**`src/ez_ax/adapters/pyautogui_adapter.py`**:

- `_capture_screenshot(self, key)`: 기존 blanket-except 제거. pyautogui 예외를 `ScreenCaptureUnavailable` 로 re-raise (원 예외 `__cause__` 보존). 호출자에서 decide.
- `_validate_bounds(self, x, y)` (신규): `pyautogui.size()` 와 비교, 범위 밖이면 `ClickCoordinatesOutOfBounds`.
- `execute(self, request)`:
  - x/y 있으면 `_validate_bounds` 선행.
  - `pyautogui.click(x, y)` 후 `pyautogui.position()` 읽어서 `(x, y)` 와 허용오차 ±2px 내 일치 확인; 아니면 `ClickExecutionUnverified`.
  - `_gather_evidence` 호출 — 예외 발생 시 FlowError 로 승격(adapter 계약에 따름) 또는 원 `ExecutionTransportError` 그대로 상향.
- `preflight(self)` (신규): (a) `moveTo(현재 위치, duration=0)` 호출 후 position 재확인 (± 오차), (b) `screenshot()` 1회 호출 후 PIL Image 수신 확인 + non-zero size. 각 실패는 해당 typed 에러로.

**`src/ez_ax/graph/pyautogui_cli_entrypoint.py`**:
- `main()` 시작 직후: `adapter = PyAutoGUIAdapter(run_root=...)` 생성 후 `adapter.preflight()`. `ExecutionTransportError` 잡아서 stderr 에 메시지 + `sys.exit(2)`.

### 3.3 Screen-Bounds Rule

Validation: `0 <= x < size.width` and `0 <= y < size.height`. 경계 위반 시 `ClickCoordinatesOutOfBounds(x=x, y=y, width=w, height=h)`.

### 3.4 Post-Click Verification Rule

- click 직후 `time.sleep(0.05)` 대기(OS 이벤트 flush) 후 `pyautogui.position()` 확인.
- 허용오차 ±2px (display DPI / 애니메이션 잔여 허용).
- 불일치 시 `ClickExecutionUnverified(expected=(x,y), actual=(px,py))`.

### 3.5 Marker Registration

`pyproject.toml`:

```toml
[tool.pytest.ini_options]
markers = [
  "real: requires OS permissions (Accessibility + Screen Recording) and a real display",
]
addopts = "-m 'not real'"
```

기존 `testpaths`, `asyncio_mode` 유지. `uv run pytest -m real` 으로 명시 실행 시만 real 테스트 수행.

## 4. Edge Cases & Error Handling

### 4.1 정상 경로

| 시나리오 | 기대 | 처리 |
|---------|------|------|
| 권한·환경 OK | preflight 통과 → 12 미션 진행 | 기존 로직 그대로 |
| click 좌표 중앙 | click 발생, position 일치 | 검증 통과 |
| screenshot 성공 | PNG 파일 실존 | evidence_ref 정직 |

### 4.2 경계 / 오류

| 시나리오 | 위험 | 대응 |
|---------|------|------|
| Accessibility 권한 없음 | click 무반응, evidence 거짓 | `preflight` 에서 `AccessibilityPermissionDenied` → exit 2 |
| Screen Recording 권한 없음 | screenshot 실패, evidence 거짓 | `preflight` 에서 `ScreenCapturePermissionDenied` |
| 실행 중간 screenshot 실패 | evidence_ref 거짓 | `ScreenCaptureUnavailable` → FlowError 승격, 미션 실패 보고 |
| 좌표가 화면 밖 | 무해하지만 click 발생 안 함 | `ClickCoordinatesOutOfBounds` → 미션 실패 |
| click 후 position 불일치 | silent no-op | `ClickExecutionUnverified` → 미션 실패 |
| pyautogui.position 자체가 실패 | 극히 드뭄 | `AccessibilityPermissionDenied` 로 분류 (읽기도 권한 요구되는 경우 있음) |

## 6. Guardrails

### Soft (project)

- 본 PRD 범위 = **adapter + entrypoint + test + docs**. 다른 경로(orchestration graph, evidence envelope) 는 손대지 않음.
- 한 커밋 = 한 문제. 5 커밋 예상: (C1) errors.py 추가, (C2) adapter hardening + 단위 테스트 조정, (C3) entrypoint preflight 호출, (C4) real integration test + marker, (C5) README/AGENTS 권한 문서화.
- preflight 실패 메시지는 사람이 읽어서 "어느 권한이 어느 앱에 필요한지" 바로 이해 가능해야 함.

### Hard (prohibitions)

- 런타임 LLM inference 도입 금지.
- Playwright/CDP/Chromium 을 execution backend 로 도입 금지.
- `except Exception:` silent-swallow 재도입 금지 (드리프트 재발 방지).
- evidence_ref 를 "파일이 없을 수도 있는" 의미로 약화 금지 — 항상 실존 보장.
- Released ceiling (`runCompletion`) 확장/축소 금지.
- pyautogui 이외의 execution library (pynput 등) 도입 금지 — adapter 단일성 유지.
