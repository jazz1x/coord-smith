## 1. Problem Definition

### 1.1 Current State (As-Is)

ez-ax는 12개 미션이 모두 released되고 703개 테스트가 통과하지만,
production-level 완성에는 4개의 구조적 갭이 남아 있다.

**갭 1 — Ghost Ref + Evidence Priority Enforcement 미적용**
`_FALLBACK_REFS`의 12개 미션 전부에 `evidence://text/fallback-reason`이 있지만
`_gather_evidence()`는 이 ref에 대응하는 text 파일을 **전혀 쓰지 않는다.**
검증 함수 `validate_action_log_evidence_refs_resolvable()`도
`kind != "action-log"`이면 skip하므로 ghost ref는 검출되지 않는다.

결과: standalone 실행 시 `ExecutionResult.evidence_refs`에 존재하지 않는 text
증거 ref가 포함돼 있다. `enforce_evidence_priority()`는 이 ghost ref를 보고
`"text"` (priority index 1)를 highest로 판정 — 실제로는 action-log / screenshot만
생성됐음에도 엔진이 자신의 증거 품질을 과장한다.

추가로, `enforce_evidence_priority()` 자체가 그래프 결정점에서 호출되지 않아
PRD가 요구하는 "typed evidence is required for released-scope decisions" 강제가
런타임에 없다.

**갭 2 — Blocking `time.sleep` in async context**
`PyAutoGUIAdapter`의 `_await_post_click_signal` (interval 폴링) 및
`_POST_CLICK_SETTLE_SECONDS` (3곳)이 `time.sleep`을 사용한다. 이 메서드들은
`async def execute()`를 통해 LangGraph async 노드에서 호출되므로 event loop을
블로킹한다.

**갭 3 — e2e 테스트 커버리지 불완전**
`tests/e2e/test_released_path_e2e.py`는 기본 경로만 커버한다.
다음 경로가 미검증이다:
- click recipe (coord 기반) 실행 경로
- evidence priority enforcement 게이트 경로
- CLI entrypoint exit code (0 / 1 / 2 / 3)

**갭 4 — `ExecutionResult.evidence_refs` 경계 검증 없음**
`ExecutionResult.evidence_refs: tuple[str, ...]`는 raw string tuple이다.
`execute_within_scope()`가 내부에서 검증하지만, Adapter가 반환하는 시점
(Protocol 경계)에서 invalid ref가 들어와도 타입 시스템이 감지하지 못한다.

### 1.2 Impact

해소하지 않으면:
- Evidence truth 계층이 코드에 존재하지만 런타임에서 무의미하다 (dead invariant).
- 고부하 또는 느린 post-click polling 시 async 노드가 수초간 블로킹된다.
- e2e 검증 없이 click recipe 경로를 production에 배포하면 silent failure 가능성이 있다.

## 3. Non-Goals

이 PRD에서 다루지 않는 항목:

- **MCP transport server 구현** — OpenClaw가 MCP를 사용하지 않음. `config/mcp_stdio.py`는 scaffold로 유지하되 구현 대상 아님.
- **release ceiling 확장** — `runCompletion` 이상 미션 추가는 이 PRD 범위 밖.
- **OpenClaw 내부 구현 변경** — ez-ax는 OpenClaw 경계 밖을 수정하지 않음.
- **dom / text / clock evidence 수집 로직** — ez-ax는 이 타입들을 생성하지 않음. 수신한 refs의 검증만 수행.
- **새 ExecutionAdapter 구현체 추가** — `PyAutoGUIAdapter`가 유일한 released adapter.

## 4. Design Decision

### Alternative A: Evidence Enforcement + Async Fix + e2e 선행 (선택)

기존 Python 스킬 6개를 모두 적용하여 현재 코드베이스 위에서 4개 갭을
순차적으로 해소한다. OpenClaw transport 계층은 현행 CLI subprocess 방식을 유지한다.

- **Approach**: 갭 4 → 갭 2 → 갭 1 → 갭 3 순서로 처리.
  각 갭은 독립 커밋으로 분리한다.
- **Pros**: 즉시 실행 가능; 기존 Protocol 구조 유지; Python 스킬 적용점 명확;
  e2e 통과로 완료 기준 충족
- **Cons**: OpenClaw 실 연동은 미검증 — CLI subprocess 동작만 보장됨
- **Suitable situation**: 현재 상태 (OpenClaw 연동 일정 미정, 코드 품질 우선)

### Alternative B: MCP Transport Server 구현

OpenClaw가 ez-ax를 MCP 서버로 호출할 수 있도록 `McpStdioAdapter` 구현.

- **Approach**: `langchain_mcp_adapters` 또는 raw `mcp` SDK로 stdio 서버 구현
- **Pros**: 미래 연동 아키텍처 준비
- **Cons**: OpenClaw가 MCP를 사용하지 않음 — 현재 불필요; 복잡도 증가
- **Suitable situation**: OpenClaw MCP 연동이 확정됐을 때

### Selection: Alternative A

**근거**: OpenClaw MCP 연동 없음이 확인됨 → Alternative B 기각.
현재 transport는 CLI subprocess로 이미 동작 중. 4개 갭은 모두 현행 코드베이스
내에서 해소 가능하며, Python 스킬 6개(error-handling, performance-optimization,
testing-patterns, type-safety, design-patterns, anti-patterns)가 각 갭에 직접 적용된다.

**유효 조건**: OpenClaw가 MCP 또는 다른 IPC 방식으로 전환되면 Alternative B 재검토.

**참조 자산**: `pyautogui-adapter-hardening-pattern` (harnish) —
preflight + typed exceptions 패턴을 이 PRD가 계승함.

### 5.2 Core Data Model

**갭 4 — ExecutionResult 경계 검증**

```python
# src/ez_ax/adapters/execution/client.py

@dataclass(frozen=True, slots=True)
class ExecutionResult:
    mission_name: str
    evidence_refs: tuple[str, ...]

    def __post_init__(self) -> None:
        if not isinstance(self.mission_name, str) or not self.mission_name.strip():
            raise ValidationError("ExecutionResult.mission_name must be non-empty string")
        if not isinstance(self.evidence_refs, tuple):
            raise ValidationError("ExecutionResult.evidence_refs must be a tuple")
        for ref in self.evidence_refs:
            parse_released_evidence_ref(ref)  # raises ValueError on invalid ref
```

**갭 1 — Ghost Ref 제거 + Evidence Priority Gate**

`_FALLBACK_REFS`에서 `evidence://text/fallback-reason` 제거:
```python
# src/ez_ax/adapters/pyautogui_adapter.py
# before (12개 미션 전부):
# ("evidence://screenshot/...", "evidence://text/fallback-reason", "evidence://action-log/...")
# after:
# ("evidence://screenshot/...", "evidence://action-log/...")
```

standalone evidence 최고 우선순위: `"action-log"` (index 3) — 실제 쓴 것만 claim.

```python
# src/ez_ax/evidence/envelope.py

def enforce_evidence_priority_gate(
    *,
    mission_name: str,
    evidence_refs: tuple[str, ...],
    minimum_priority: EvidencePriority = "action-log",
) -> None:
    """Released-scope 결정 전에 증거 우선순위 게이트를 통과시킨다.

    Raises:
        ValidationError: refs가 minimum_priority보다 낮은 경우
    """
    if not evidence_refs:
        raise ValidationError(f"Mission '{mission_name}': evidence_refs must be non-empty")
    highest = enforce_evidence_priority(set(evidence_refs))
    priority_rank = {k: i for i, k in enumerate(EVIDENCE_PRIORITY_ORDER)}
    if priority_rank[highest] > priority_rank[minimum_priority]:
        raise ValidationError(
            f"Mission '{mission_name}': highest evidence kind '{highest}' "
            f"is below minimum '{minimum_priority}'"
        )
```

### 5.3 Interface/API Design

**갭 2 — PyAutoGUIAdapter async 시그니처 변경 없음**

`execute()`는 이미 `async def`이므로 내부 `time.sleep` → `asyncio.sleep`만 교체.
외부 인터페이스 변경 없음.

```python
# 변경 전
time.sleep(_POST_CLICK_SETTLE_SECONDS)

# 변경 후
await asyncio.sleep(_POST_CLICK_SETTLE_SECONDS)
```

`_await_post_click_signal`은 sync이므로 `_async_await_post_click_signal`으로 분리:

```python
async def _async_await_post_click_signal(
    self, *, mission: str, signal: PostClickSignal
) -> None:
    ...
    while elapsed < signal.timeout:
        ...
        await asyncio.sleep(interval)
```

## 7. Edge Cases & Error Handling

| 시나리오 | 기대 동작 | 처리 전략 |
|---------|----------|---------|
| `ExecutionResult(evidence_refs=("bad-ref",))` | `ValidationError` | `__post_init__`에서 즉시 raise |
| ghost ref 제거 후 evidence_refs = (action-log, screenshot)만 남음 | gate passes: action-log (index 3) ≤ minimum action-log (index 3) | ✅ |
| evidence_refs가 coordinate-only | `ValidationError` (gate 실패) | `enforce_evidence_priority_gate` minimum="action-log" |
| post_click_signal timeout 중 asyncio cancel | `ImageWaitTimeout` 유지 | `asyncio.sleep` 중단 시 루프 탈출 후 raise |
| `_async_await_post_click_signal` 호출 경로가 sync인 경우 | 컴파일 에러로 조기 발견 | mypy strict가 잡음 |

**python-error-handling 스킬 적용 포인트**:
- `enforce_evidence_priority_gate` — `ValidationError` 계층 일관성 확인
- `ExecutionResult.__post_init__` — `ValidationError` vs `TypeError` 경계 명확화

## 9. Guardrails

### Project Guardrails (soft)

- `enforce_evidence_priority_gate` 호출 시 minimum은 `"action-log"` 이상으로 유지
- async 분리된 `_async_await_post_click_signal`은 sync 경로에서 호출 금지
- `ExecutionResult.__post_init__` 검증은 `parse_released_evidence_ref`를 재사용
  (중복 검증 로직 금지)

### Prohibitions (hard)

- `runCompletion` 이상 미션 확장 금지 — e2e 테스트가 새 미션을 추가하면 즉시 중단
- `time.sleep` 재도입 금지 — pyautogui_adapter.py에서 blocking sleep은 anti-pattern
- `enforce_evidence_priority_gate`를 bypass하는 `execute_within_scope` 래퍼 금지
- `dom` / `text` / `clock` evidence ref를 ez-ax 내부에서 생성하는 것 금지
  (이 타입들은 OpenClaw만 생성 가능 — ez-ax는 검증만 수행)
