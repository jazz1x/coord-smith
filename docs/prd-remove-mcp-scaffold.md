## 1. Problem Definition

### 1.1 Current State (As-Is)

ez-ax는 `src/ez_ax/config/mcp_stdio.py` (157줄), `src/ez_ax/config/mcp_stdio_cli.py` (95줄), `tests/unit/test_mcp_stdio_cli.py` (6 tests) 를 **inactive scaffold**로 유지하고 있다. 이들은:

- **활성 released path 어디서도 import되지 않는다.** `pyautogui_cli_entrypoint → run_released_scope_from_argv_env` 체인은 MCP 심볼을 전혀 참조하지 않는다.
- **외부 의존성에서 이미 분리되었다.** `mcp` 런타임 의존성은 commit `0dbeae72`에서 제거됨.
- **docstring에 "Inactive scaffold" 명시가 추가되었지만** (commit `41a32bd`), 코드 자체는 여전히 활성 src 트리에 존재한다.

이전 PRD (`docs/history/prd-production-hardening.md`)는 "scaffold로 유지하되 구현 대상 아님"으로 명시했지만, 사용자는 **MCP를 영구적으로 채택하지 않기로 결정**했다.

### 1.2 Target State (To-Be)

- `src/ez_ax/config/mcp_stdio.py` 와 `src/ez_ax/config/mcp_stdio_cli.py` 가 존재하지 않는다.
- `tests/unit/test_mcp_stdio_cli.py` 가 존재하지 않는다.
- `docs/prd.md` 의 Non-Goals 섹션에 "MCP transport는 영구히 채택되지 않는다" 가 명시된다.
- 자율형 에이전트(OpenClaw)가 YAML 레시피를 작성 → CLI 호출 → artifacts 확인 흐름이 그대로 동작한다 (acceptance test 6개 통과).
- 회귀: pytest passing count는 727 → 721 (제거된 6 tests만큼 감소, 기존 release path 변동 없음). mypy strict / ruff clean 유지.

### 1.3 Business/Technical Impact

해소 안 하면:
- 새로운 기여자나 자율 에이전트가 src 트리를 탐색할 때 "config/mcp_stdio.py 는 무엇인가?" 매번 판단 비용을 지불한다.
- 미래 변경에서 "MCP는 그대로 두라" vs "MCP도 같이 갱신하라" 같은 지시가 모호해진다.
- "inactive scaffold" 라는 docstring 문구는 유지해야 할 코드와 죽은 코드 사이의 회색지대를 만든다.

해소하면:
- 활성 src의 의도가 100% 명료해진다 — 모든 파일은 released path에 기여한다.
- 미래 MCP 채택 결정이 다시 발생하면 git history(commit `0dbeae72` 이전)에서 형태를 복구 가능하다.

## 2. Design Decision

### Alternative A: Hard delete

- **Approach**: `mcp_stdio.py`, `mcp_stdio_cli.py`, `test_mcp_stdio_cli.py` 3개 파일을 단일 커밋으로 삭제. PRD/Non-Goals를 동기화.
- **Pros**: 인지부담 제로화, -244 LOC, 외부 import 0건이라 안전, 회복은 git history에서 가능.
- **Cons**: MCP 부활 시 재구현 비용. 단, 동일 모듈을 커밋 hash로 부활 가능하므로 비용 ≈ 0.
- **Suitable situation**: 사용자가 MCP를 영구히 채택하지 않기로 결정한 경우 (현재 상황).

### Alternative B: Status quo + reinforced docstring

- **Approach**: 현재처럼 inactive scaffold docstring 유지. 추가로 import-warning 또는 deprecation marker 부착.
- **Pros**: 학습 자산이 working tree에 남음.
- **Cons**: 사용자가 직접 거부한 상태와 모순. 인지부담 동일하게 잔존. 이미 commit `41a32bd`에서 시도되었으나 충분치 않다고 판단됨.
- **Suitable situation**: MCP 채택 가능성이 0이 아닐 때.

### Selection: Alternative A

- **Rationale**: 외부 import 0건 + 의존성 이미 정리 + 사용자 명시적 영구 거부 + git history로 회복 가능. 현재 보유 자산과 향후 회복 비용을 모두 고려해도 Alt B 대비 명백한 이득.
- **Rejected alternatives and reasons**: Alt B는 사용자의 결정과 모순 (이미 거부된 시도).
- **Conditions under which this decision is valid**: OpenClaw가 향후 MCP transport로 전환을 결정하지 않는 한 유효. 만약 결정되면 commit `0dbeae72` 이전의 형태를 git에서 복원하여 재시작.
- **Referenced existing assets**: `[pattern] ez-ax click recipe flow pattern` (CLI subprocess transport가 활성 채널임을 재확인).

## 7. Guardrails

### Project Guardrails (soft — warn on violation)

- **활성 src/ 디렉터리에 dead-code 신규 추가 금지.** "scaffold for future" 라는 명목의 코드는 별도 문서나 PRD 본문에만 보존하고, src에는 active 코드만 둔다.
- **삭제 단일 커밋 원칙.** 3개 파일 삭제 + 문서 동기화는 하나의 커밋으로 묶는다 (lin나 부분 삭제로 인한 중간 상태 회피).

### Prohibitions (hard — stop immediately on violation)

- **Released path 코드 수정 금지.** `src/ez_ax/graph/`, `src/ez_ax/adapters/`, `src/ez_ax/evidence/`, `src/ez_ax/config/click_recipe.py`, `src/ez_ax/config/released_inputs.py`, `src/ez_ax/missions/` 의 어떤 파일도 이 PRD 범위에서 변경하지 않는다.
- **Acceptance test 수정 금지.** `tests/e2e/test_agent_acceptance.py` 는 이번 변경의 회귀 시그널이므로 함께 수정해선 안 된다.
- **PRD 영구 거부 문구 우회 금지.** Non-Goals 섹션의 "MCP transport — 영구히 채택하지 않음" 문구는 향후 다른 PRD에서 부활시킬 때만 변경 가능. 일반 cleanup PR에서 임의로 약화 금지.
